# Админ-команды бота «Бро Кколи»

Доступ ко всем командам ниже есть **только у пользователя**, чей Telegram ID указан в `.env` как **`ADMIN_ID`**. Остальным бот отвечает, что доступа нет.

---

## Команды в чате (со слэшем)

### `/stock`

Открывает **панель наличия товаров** в виде инлайн-кнопок.

- кнопка с `✅` означает, что товар **в наличии**;
- кнопка с `❌` означает, что товар **нет в наличии**;
- нажатие по товару **переключает статус**.

Mini App читает наличие из `web/stock.json`. Бот теперь пытается **автоматически закоммитить и запушить** этот файл в репозиторий `web/`, чтобы GitHub Pages показал новое состояние без ручной публикации.

Если `git push` не сработает \(например, не настроена авторизация или remote\), бот сообщит админу об ошибке отдельным сообщением.

### `/stats`

Показывает **статистику по подтверждённым заказам**:

- общее число заказов со статусом «подтверждён»;
- разбивка по целям покупки: **Быт**, **Спорт**, **Хадж** (сколько заказов в каждой категории).

Используется, чтобы видеть объём продаж по сегментам.

---

### `/padd`

**Создаёт или обновляет промокод** в базе (код, процент скидки, имя амбассадора/партнёра).

**Формат:**

```text
/padd КОД СКИДКА ИМЯ
```

**Пример:**

```text
/padd MEKKA10 10 Абдулла
```

- **КОД** — латиница/цифры, в базе хранится в верхнем регистре.
- **СКИДКА** — целое число от 1 до 99 (проценты).
- **ИМЯ** — произвольный текст (можно несколько слов); если не нужно — можно оставить пустым, указав только код и скидку не получится (минимум два аргумента после команды: код и число).

Если код уже существует, запись **обновляется** (новый процент и имя).

После добавления промокода его нужно **прописать в Mini App** (`web/promos.json` и/или `PROMO_CODES` в `script.js`), иначе кнопка «Применить» в приложении не найдёт код — **итоговая проверка всё равно на сервере в БД**.

---

### `/plist`

Выводит **список всех промокодов** в виде **инлайн-кнопок** (код и процент).

Нажатие на кнопку отправляет **тот же отчёт**, что и `/pinfo` для этого кода (см. ниже).

---

### `/pinfo`

**Отчёт по одному промокоду** (текстом в чат).

**Формат:**

```text
/pinfo КОД
```

**Пример:**

```text
/pinfo MEKKA10
```

В отчёте: партнёр, процент скидки, число заказов с этим кодом, сумма к оплате по заказам (оборот), сумма выданных скидок, напоминание про ручной расчёт доли амбассадору.

---

### `/pdel`

**Удаляет промокод** из базы.

**Формат:**

```text
/pdel КОД
```

**Пример:**

```text
/pdel MEKKA10
```

Если кода нет — бот сообщит об этом.

---

## Инлайн-действия (не команды со слэшем)

### Кнопки после `/plist`

Callback `admprm:КОД` — при нажатии на строку из списка промокодов приходит **тот же отчёт**, что и у `/pinfo КОД`.

---

### Кнопки под сообщением о новом заказе

Когда клиент оформляет заказ из Mini App, админу приходит уведомление с кнопками (callback `admord:…`):

| Действие | Смысл |
|----------|--------|
| **Подтвердить** | Заказ переводится в статус «подтверждён» — **учитывается** в `/stats`. |
| **Отменить** | Заказ отменён — в статистику не входит. |
| **Пометить заказ** | Статус «подготовка» — в статистике **ещё не учитывается**, пока не нажмёте «Подтвердить». |

После выбора статуса сообщение обновляется, клавиатура меняется (чтобы не нажимать повторно без необходимости).

## Краткая таблица

| Команда | Назначение |
|---------|------------|
| `/stock` | Включить/выключить наличие товара |
| `/stats` | Статистика подтверждённых заказов по целям |
| `/padd` | Добавить/обновить промокод |
| `/plist` | Список промокодов (кнопки → отчёт) |
| `/pinfo` | Отчёт по одному коду |
| `/pdel` | Удалить промокод |
